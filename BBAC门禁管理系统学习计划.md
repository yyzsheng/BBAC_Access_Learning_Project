# BBAC 门禁管理系统零基础项目式学习计划

## 1. 学习目标

本计划的目标不是由 AI 代替学习者完成项目，而是通过持续讲解、练习、作业、代码评审和验收，让学习者最终能够独立完成并维护一个内部运行的 Web 系统。

项目最终需要做到：

- 管理约 20,000 名正式工、三方和访客。
- 管理工号、NFC 卡号、组织、属地区域和逻辑门禁点。
- 通过 Excel/CSV 导入刷卡记录。
- 查询进场、出场、在场人员和历史记录。
- 管理访客、重点人员、权限和审批记录。
- 提供登录、角色权限、审计日志和总览大屏。
- 部署到 Linux 服务器并通过 Docker 运行。
- 不涉及门锁、控制器、读卡器和线路等物理设备。

## 2. 时间安排

- 总周期：32 周。
- 每周学习：6 天，保留 1 天休息或补课。
- 每天学习：约 5 小时。
- 预计总投入：850 至 950 小时。

每天建议节奏：

1. 60 分钟：学习新知识，理解用途和原理。
2. 60 分钟：跟随小示例练习。
3. 120 分钟：独立完成当天项目任务。
4. 30 分钟：运行检查、测试和排错。
5. 30 分钟：记录笔记、错误和 Git 提交。

## 3. 教学与协作规则

- 先解释为什么，再讲怎样实现。
- 学习者亲手输入核心代码，不直接复制完整答案。
- 遇到错误时保留完整错误信息，先分析再修改。
- AI 提供知识讲解、任务拆分、代码骨架、提示、评审和验收。
- 每个功能完成后运行测试并提交 Git。
- 每周进行知识问答、独立作业、项目检查和复盘。
- 核心验收不通过时先补课，不盲目赶进度。
- 现有单文件 Demo 只作为需求和视觉参考，不复制到正式项目。

## 4. 第一阶段：Python 与编程基础（第 1-5 周）

### 第 1 周：环境、终端与 Git

学习：macOS 终端、路径、进程、端口、VS Code、Git、Python 虚拟环境。

项目任务：认识项目目录，运行第一个 Python 文件，完成第一次学习提交。

验收：能独立进入目录、运行 Python、查看 Git 状态并完成提交。

### 第 2 周：Python 基础语法

学习：变量、字符串、数字、布尔值、列表、字典、集合、条件和循环。

项目任务：编写命令行人员清单，支持新增姓名、工号、卡号和人员类型。

验收：能独立编写条件和循环，能解释列表与字典的区别。

### 第 3 周：函数、模块、异常和文件

学习：函数参数、返回值、作用域、模块、`try/except`、JSON 文件读写。

项目任务：拆分人员管理函数，将人员保存到 JSON，重启程序后恢复数据。

验收：函数职责清楚，能够处理文件不存在和输入格式错误。

### 第 4 周：类型、类与数据校验

学习：类、对象、类型注解、`dataclass`、Pydantic 基础。

项目任务：建立 `Person`、`Card`、`AccessRecord` 模型并校验必填字段、时间和进出方向。

验收：能根据业务字段设计模型，能读懂数据校验错误。

### 第 5 周：测试与命令行小项目

学习：Pytest、断言、边界情况、异常情况、断点调试。

项目任务：完成命令行版人员与刷卡记录系统，支持 CSV 导入和当天进出查询，编写至少 10 个测试。

验收：测试通过，能解释数据从文件进入程序再生成统计的过程。

## 5. 第二阶段：网页与 Vue 前端（第 6-13 周）

### 第 6 周：HTML 与 CSS

学习：HTML 结构、表格、表单、CSS 盒模型、Flex、Grid、响应式布局和开发者工具。

项目任务：手写人员管理静态页面，适配 1366×768 和 1920×1080。

### 第 7 周：JavaScript 基础

学习：变量、函数、数组、对象、`map`、`filter`、`find`、DOM 和事件。

项目任务：在网页中新增、搜索和筛选人员。

### 第 8 周：异步与 HTTP

学习：HTTP、JSON、GET、POST、PUT、DELETE、Promise、`async/await` 和错误处理。

项目任务：从模拟接口加载人员，处理等待、成功和失败状态。

### 第 9 周：TypeScript

学习：基础类型、接口、联合类型、可选字段、泛型和类型缩小。

项目任务：定义人员、卡片和通行记录类型，将练习页面迁移到 TypeScript。

### 第 10 周：Vue 3 基础

学习：组件、模板、`ref`、`reactive`、`computed`、Props、Emits 和组合式 API。

项目任务：创建 Vue 项目，将人员页面拆成筛选栏、表格和表单组件。

### 第 11 周：路由与状态管理

学习：Vue Router、Pinia、页面布局、登录状态和路由守卫。

项目任务：建立登录、总览、人员、访客和通行记录页面路由。

### 第 12 周：Element Plus 管理页面

学习：表格、分页、表单、弹窗、校验、加载状态、空状态和二次确认。

项目任务：完成人员管理页面的查询、分页、重置、新增、编辑和停用。

### 第 13 周：ECharts 总览

学习：图表数据结构、柱状图、分类统计、响应式图表和统计口径。

项目任务：完成日期、MRA/PT1/PT2、进场、出场、在场人数和一周趋势页面。

阶段验收：能独立创建 Vue 页面和组件，能解释浏览器请求后台的过程。

## 6. 第三阶段：MySQL 数据库（第 14-16 周）

### 第 14 周：SQL 基础

学习：表、行、列、增删改查、条件、排序、分页、分组和聚合。

项目任务：建立人员和卡片练习表，完成常见查询和分类统计。

### 第 15 周：关系与数据库设计

学习：主键、外键、唯一约束、一对多、多对多和数据库规范化。

项目任务：设计组织、人员、卡片、区域、门点、权限组和刷卡记录关系图。

### 第 16 周：索引与事务

学习：索引、组合索引、事务、回滚、查询计划和分页性能。

项目任务：为工号、卡号、刷卡时间和厂区建立索引，模拟批量记录查询。

阶段验收：能够独立编写关联查询，并解释主键、外键、索引和事务。

## 7. 第四阶段：FastAPI 后端（第 17-22 周）

### 第 17 周：FastAPI 基础

学习：路由、请求方法、参数、请求体、响应模型、状态码和接口文档。

项目任务：创建 FastAPI 项目，实现健康检查、人员查询和新增接口。

### 第 18 周：SQLAlchemy 与 Alembic

学习：ORM、数据库会话、查询、事务、Service 分层和数据库迁移。

项目任务：连接 MySQL，建立人员、卡片和组织模型并完成增删改查。

### 第 19 周：登录与认证

学习：密码哈希、JWT、Access Token、Refresh Token、Cookie 和认证流程。

项目任务：完成登录、退出、当前用户和登录失效处理。

### 第 20 周：角色权限与审计

学习：RBAC、接口权限、逻辑删除和审计日志。

项目任务：建立管理员、只读用户和审计用户，记录关键管理操作。

### 第 21 周：Excel/CSV 导入

学习：文件上传、字段映射、数据清洗、批量写入、去重和错误报告。

项目任务：上传刷卡文件、预览、校验、导入并下载错误明细。

### 第 22 周：后端测试

学习：接口测试、测试数据库、Fixture、权限测试和导入测试。

项目任务：覆盖登录、人员、卡片、文件错误、重复记录和无权限操作。

阶段验收：能独立编写分页接口、数据库迁移和接口测试。

## 8. 第五阶段：正式业务功能（第 23-30 周）

### 第 23 周：前后端连接

完成 Axios 请求封装、登录、令牌刷新、统一错误提示和真实分页。

### 第 24 周：组织、人员与卡片

完成组织树、正式工/三方/访客、工号、卡片绑定、停用、换卡和人员导入。

### 第 25 周：区域与属地权限

完成 MRA、PT1、PT2、内部区域、逻辑门禁点、权限组和人员临时权限，不实现物理设备控制。

### 第 26 周：访客与重点人员

完成访客申请、被访问人部门、到访区域、访客卡号、授权记录和重点人员管理。

### 第 27 周：刷卡记录

完成导入、去重、分页，以及姓名、工号、卡号、日期、区域、门点和方向筛选。

### 第 28 周：总览统计

完成总通行、进场、出场、在场人数、人员分类、一周趋势和三厂区统计，全部从数据库计算。

### 第 29 周：审批、导出与审计

完成已通过授权记录、查询导出、操作审计和关键操作二次确认。

### 第 30 周：系统质量

完成前端测试、后端测试、关键流程测试、2 万人员模拟和大量刷卡记录性能检查。

阶段验收：从登录到导入、查询、授权、统计和审计的完整流程可运行。

## 9. 第六阶段：部署与运行（第 31-32 周）

### 第 31 周：Docker 与 Linux

学习：Linux 用户和权限、进程、日志、Docker、Docker Compose、Nginx 和环境变量。

项目任务：容器化前端、后端和 MySQL，在测试服务器一条命令启动。

### 第 32 周：备份、恢复与验收

学习：MySQL 备份恢复、HTTPS、健康检查、发布和回滚。

项目任务：配置每日备份，完成恢复演练，编写部署和操作手册，连续试运行两周。

最终验收：

- 能独立启动前端、后端和数据库。
- 能解释主要目录和代码职责。
- 能独立增加字段、接口和查询条件。
- 能定位常见前端、接口和数据库错误。
- 能部署新版本并恢复数据库。
- 系统可供少量内部管理员使用。

## 10. 每周验收制度

每周总分 100 分：

- 知识问答：25 分。
- 独立编程作业：35 分。
- 项目功能和 Git 提交：25 分。
- 学习笔记与复盘：15 分。

结果：

- 80 分以上：进入下一周。
- 60 至 79 分：补做薄弱练习后进入下一周。
- 60 分以下：重新学习关键知识。

## 11. 第一课入口

完成初始化后，以“开始第 1 周第 1 天”进入正式学习。

第一天只完成：

1. 检查 macOS、Python、Node.js、Git 和 VS Code。
2. 认识项目目录和 Git 状态。
3. 在 `backend/` 中亲手创建第一个 Python 文件。
4. 运行程序并理解每一行。
5. 填写每日学习记录。
6. 完成知识问答和第一次学习提交。
