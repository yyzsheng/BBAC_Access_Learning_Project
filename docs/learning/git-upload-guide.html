Git 上传 GitHub 学习流程

适用于本项目：BBAC_Access_Learning_Project

一句话记住

Git 上传分三步：add 选文件，commit 保存到本地历史，push 上传到 GitHub。

git add ...
git commit -m "提交说明"
git push

0. 先进入项目目录

如果终端前面显示的是 ~，说明你在家目录，不在项目里。必须先进入项目目录。

cd /Users/hongxuchai/Desktop/BBAC_Access_Learning_Project

然后检查位置：

pwd

正确时应该看到：

/Users/hongxuchai/Desktop/BBAC_Access_Learning_Project

常见错误

fatal: not a git repository (or any of the parent directories): .git

意思是：你不在 Git 项目目录里。解决方法就是先执行上面的 cd 命令。

1. 查看当前状态

git status

这个命令是每次提交前都要看的。它会告诉你：

  • 哪些文件是新建的
  • 哪些文件被修改了
  • 哪些文件已经准备提交

2. 选中要提交的文件：git add

如果你看到：

Untracked files:
  backend/access_record_cli.py
  backend/access_record_model.py

意思是：这些文件 Git 还没有跟踪，需要先 add。

git add backend/access_record_cli.py backend/access_record_model.py backend/test_access_record_model.py backend/sample_access_records.csv backend/valid_access_records.json

如果你确定当前所有改动都要提交，也可以用：

git add .

初学阶段建议先用明确文件名，比较安全。

3. 再次检查状态

git status

如果 add 成功，会看到类似：

Changes to be committed:
  new file:   backend/access_record_cli.py
  new file:   backend/access_record_model.py

这表示文件已经进入“准备提交区”。

4. 创建本地提交：git commit

git commit -m "learn: complete access record CLI exercise"

-m 后面是这次提交的说明。提交说明要简单描述你做了什么。

提交说明 适合场景
learn: add access record model 新增刷卡记录模型
learn: add access record CLI 新增命令行菜单
learn: add access record tests 新增测试

如果 commit 时看到这个

nothing added to commit but untracked files present

意思是：你还没有执行 git add，所以 Git 没有东西可以提交。

解决方法：

git add 文件名
git commit -m "提交说明"

5. 上传到 GitHub：git push

git push

这一步会把本地 commit 上传到 GitHub。

如果你没有先 commit，push 会显示：

Everything up-to-date

这不是错误，它的意思是：没有新的提交需要上传。

6. 最后确认是否干净

git status

正常完成后，应该看到类似：

Your branch is up to date with 'origin/main'.
nothing to commit, working tree clean

这表示：本地没有未提交内容，GitHub 也已经同步。

完整流程模板

cd /Users/hongxuchai/Desktop/BBAC_Access_Learning_Project
git status
git add 文件名1 文件名2 文件名3
git status
git commit -m "提交说明"
git push
git status

你当前项目常用命令

运行刷卡记录菜单

python3 backend/access_record_cli.py

运行测试

python3 -m unittest backend.test_access_record_model

检查 Python 语法

python3 -m py_compile backend/*.py

学习重点

命令 中文理解
git status 看看现在有什么变化
git add 选中要提交的文件
git commit 把选中的文件保存成本地历史记录
git push 把本地历史记录上传到 GitHub
